Magic commands: quick queries and more¶

DimCli includes a few Python magic commands which make it much easier to interrogate the API.

[1]:

!pip install dimcli -U --quiet
import dimcli
dimcli.login(username="dimensions-username@me.com", password="my-secret-password")

     |████████████████████████████████| 122kB 4.9MB/s eta 0:00:01
DimCli v0.6.1 - Succesfully connected to <https://app.dimensions.ai> (method: manual login)

Magic commands are essentially shortcuts that allow to launch queries without having to create an object using the syntax dsl = dimcli.Dsl().

They can be useful when testing things out e.g. while trying out a new query, or checking what data is available in Dimensions on a certain topic.

1. Querying with %dsl or %%dsl¶

This command allows to simply pass an API query after typing %dsl.

Single-line version

[4]:

%dsl search publications where journal.title="Nature Energy" return publications

Returned Publications: 20 (total = 901)

[4]:

<dimcli.Result object #140002153721192. Records: 20/901>

Multi-line version

You can also split the query into multiple lines, only this time you need to use the %%dsl command (two %):

[5]:

%%dsl
search publications
where year in [2013:2018] and research_orgs="grid.258806.1"
return publications[title]

Returned Publications: 20 (total = 3815)

[5]:

<dimcli.Result object #140002179358112. Records: 20/3815>

Accessing Results¶

By default the results of magic command queries are saved into a variable called dsl_last_results:

[ ]:

type(dsl_last_results)

dimcli.core.api.Result

A DimCli Result object is just a wrapper around the raw JSON data.

[6]:

print(dsl_last_results.publications[0]['title'])

Development of Organo-Dispersible Graphene Oxide via Pseudo-Surface Modification for Thermally Conductive Green Polymer Composites.

2. Loop queries with %dslloop or %%dslloop¶

This magic command automatically loops over all the pages of a results set set until all possible records have been returned.

NOTE this is a short version of the Dimcli.Dsl.query_iterative method, which takes care of timing queries appropriately and aggregating results within a single object.

Single-line version

[15]:

%dslloop search publications for "malaria AND Egypt" where year=2015 return publications

Output hidden; open in https://colab.research.google.com to view.

Multi-line version

[9]:

%%dslloop
search publications
where year in [2013:2018] and research_orgs="grid.258806.1"
return publications

Output hidden; open in https://colab.research.google.com to view.

Like before, the results of a loop query are stored into the dsl_last_results variable.

[10]:

dsl_last_results.stats

[10]:

{'total_count': 3815}

3. Retrieving documentation with %dsldocs¶

The %dsldocs command prints out information about the fields and entities available via the Dimensions Search Language.

This commands returns a tabular version of the data model documentation by using the `describe <https://docs.dimensions.ai/dsl/data.html#getting-documentation-programmatically>`__ DSL command behind the scenes.
